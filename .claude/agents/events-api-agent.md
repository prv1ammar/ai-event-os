## Endpoints to Build

### Events — /api/v1/events
```
GET    /api/v1/events                    # list with filters: status, category, year
POST   /api/v1/events                    # create event (admin/organizer only)
GET    /api/v1/events/{id}               # event detail
PUT    /api/v1/events/{id}               # update event
DELETE /api/v1/events/{id}               # soft delete (set status=archived)
GET    /api/v1/events/{id}/stats         # KPIs: visitors, exhibitors, revenue MAD, leads
GET    /api/v1/events/{id}/dashboard     # full dashboard data for organizer
```

### Exhibitors — /api/v1/exhibitors
```
GET    /api/v1/exhibitors                # list, filter by status/sector/event_id
POST   /api/v1/exhibitors                # register exhibitor
GET    /api/v1/exhibitors/{id}           # detail with booth + payment info
PUT    /api/v1/exhibitors/{id}           # update exhibitor
PUT    /api/v1/exhibitors/{id}/status    # validate/refuse exhibitor
GET    /api/v1/exhibitors/{id}/leads     # leads generated by this exhibitor
GET    /api/v1/exhibitors/{id}/offer.pdf # generate PDF commercial offer
```

## Pydantic Schema Pattern
```python
# schemas/event.py — follow this pattern for all schemas

from pydantic import BaseModel, ConfigDict
from datetime import datetime
from uuid import UUID

class EventBase(BaseModel):
    name: str
    description: str | None = None
    start_date: datetime
    end_date: datetime
    venue: str
    city: str
    country: str = "Maroc"
    capacity: int
    category: str

class EventCreate(EventBase):
    budget_mad: float | None = None

class EventUpdate(BaseModel):
    name: str | None = None
    description: str | None = None
    status: str | None = None
    # all fields optional for PATCH-style updates

class EventResponse(EventBase):
    model_config = ConfigDict(from_attributes=True)
    id: UUID
    status: str
    created_at: datetime
    updated_at: datetime

class EventStatsResponse(BaseModel):
    event_id: UUID
    total_visitors: int
    total_exhibitors: int
    total_revenue_mad: float
    total_leads: int
    confirmed_visitors: int
    occupancy_rate: float
```

## Router Pattern
```python
# routers/events.py — follow this exact pattern

from fastapi import APIRouter, Depends, HTTPException, Query
from sqlalchemy.ext.asyncio import AsyncSession
from app.core.database import get_db
from app.core.security import get_current_user
from app.schemas.event import EventCreate, EventResponse, EventStatsResponse
from app.services import event_service

router = APIRouter(prefix="/api/v1/events", tags=["Events"])

@router.get("", response_model=list[EventResponse])
async def get_events(
    status: str | None = Query(None),
    category: str | None = Query(None),
    page: int = Query(1, ge=1),
    limit: int = Query(20, ge=1, le=100),
    db: AsyncSession = Depends(get_db),
    current_user = Depends(get_current_user)
):
    return await event_service.get_all(db, status, category, page, limit)

@router.post("", response_model=EventResponse, status_code=201)
async def create_event(
    data: EventCreate,
    db: AsyncSession = Depends(get_db),
    current_user = Depends(get_current_user)
):
    return await event_service.create(db, data, current_user)
```

## Business Rules
- Event status flow: `draft` → `published` → `ongoing` → `completed`
- Booth can only be reserved if exhibitor status is `validated`
- Session capacity must not exceed room capacity
- Exhibitor `offer.pdf` must include price in MAD with VAT (20%)
- Floor plan endpoint returns booth grid with zone colors and occupancy %
